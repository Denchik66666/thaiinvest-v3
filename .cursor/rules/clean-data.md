# Чистые данные и пользователи (обязательно для агента)

Постоянные правила от владельца проекта:

1. **Никогда** не создавать тестовых пользователей, инвесторов и seed-данных без прямого разрешения пользователя словами **«создай тестового»**.
2. **Никогда** не запускать `npx prisma db seed` (и аналоги полного сида) без явного разрешения пользователя.
3. **Никогда** не дублировать существующих пользователей (не создавать вторую учётку с тем же смыслом/ролью, если запись уже есть).
4. Для проверок, e2e и скриншотов использовать **только** существующие учётки: **`admin`** (SUPER_ADMIN), **`Sam`** (OWNER), **`Sega_55RUS`** (INVESTOR). Пароли по умолчанию из `prisma/seed.ts`: `admin123` / `admin123` / `qwerty123` (не выдумывать новые без согласования).
5. Если данных не хватает — **спросить пользователя**, а не создавать данные самостоятельно.

При сомнениях — остановиться и запросить подтверждение.
