# Папка «Для анализа»

Заметки владельца продукта, чеклисты и **продуктовый канон** (что должно быть по смыслу). Код и релизы живут в корне репозитория.

## Три слоя «правды» (чтобы не путаться)

1. **Задумка и сценарии** — этот каталог: **`Подробный для ИИ Copilot.txt`**, **`шаблон для ИИ.txt`**, **`ГЛОССАРИЙ.txt`**.
2. **Как работает сейчас в приложении** — исходники **`app/`**, **`lib/`** и при необходимости **`SESSION_LOG.md`** (инженерные разборы).
3. **Версии и обзор репозитория** — **`CHANGELOG.md`**, **`PROJECT_AUDIT.md`**, **`API_AUDIT.md`**, **`.cursor/rules/`**.

Если (2) не совпадает с (1) — это либо бэклог «догнать код», либо устаревшая формулировка в тексте (1). Таблица API в **`API_AUDIT.md`** должна совпадать с **`app/api/**/route.ts`**.

## Файлы в этой папке

| Файл | Назначение |
|------|------------|
| **`ГЛОССАРИЙ.txt`** | Короткие определения ролей и терминов для агента. |
| **`Подробный для ИИ Copilot.txt`** | Полная продуктово-техническая спека. |
| **`шаблон для ИИ.txt`** | Живая бизнес-история и сценарии. |
| **`Агент_сам_что_сделать.txt`** | Что агент может в терминале без ваших паролей. |
| **`ОДНА_БАЗА_проверка_и_миграции.txt`** | Одна база: Vercel / локалка / `db:verify`. |
| **`Без_программиста_что_сделать_в_браузере.txt`** | Supabase и строки подключения в браузере. |
| **`ГДЕ_КЛИКАТЬ_Vercel_переменные_проекта.txt`** | Куда нажать в UI Vercel для переменных. |
| **`дизайн_UI_единый_стиль.txt`** | Краткая памятка по `.thai-*` и маршрутам; сверять с `.cursor/rules`. |
| **`archive/`** | Исторические заметки (снимок статуса, старый план, разбор старого Supabase). |

## Рекомендация

Не дублировать длинные техспеки: одна актуальная таблица HTTP — в **`API_AUDIT.md`**; история версий — в **`CHANGELOG.md`**.
