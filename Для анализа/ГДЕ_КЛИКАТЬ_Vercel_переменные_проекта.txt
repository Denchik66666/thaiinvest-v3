Почему «сам не найду» и что делать за 2 минуты
============================================

Агент в Cursor НЕ имеет доступа к твоему браузеру, паролю Vercel и
аккаунту Supabase. Он не может открыть сайт и нажать кнопки за тебя.
Он может: править код в папке проекта, запускать команды (npm, prisma),
писать инструкции.

Чтобы не путаться в Vercel: переменные НЕ в «команде», а в ПРОЕКТЕ
================================================================

1. Зайди на https://vercel.com/dashboard

2. В списке найди КАРТОЧКУ ПРИЛОЖЕНИЯ (проект), где лежит ThaiInvest —
   это НЕ страница «Team Settings» сверху.

3. Нажми на НАЗВАНИЕ этого проекта — откроется обзор деплоев.

4. Сверху открой вкладку **Settings** (уже настроек ЭТОГО проекта).

5. Слева в меню выбери **Environment Variables**.

6. Там должны быть (или добавь):
   DATABASE_URL
   DIRECT_URL
   JWT_SECRET
   NEXT_PUBLIC_APP_URL

7. Для каждой переменной выбери окружение: **Production** (и при
   необходимости **Preview**).

Страница «Team Settings → Environment Variables» — это ОБЩИЕ переменные
на всю команду; для одного сайта обычно достаточно пункта 4–5 выше.

Проверка после правок
=====================

На своём ПК в папке проекта в терминале:
  npm run db:verify

Строки DATABASE_URL / DIRECT_URL в .env должны быть из ТОГО ЖЕ проекта
Supabase, что и в Vercel (см. файл ОДНА_БАЗА_проверка_и_миграции.txt).

Если подключишь в Cursor Playwright MCP или Chrome DevTools MCP —
агент сможет подсказывать по скриншоту «кликни сюда», но вход в аккаунт
всё равно твой.
